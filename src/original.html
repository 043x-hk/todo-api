Introduction

In this activity, you will be using the Todo API to perform CRUD operations using Postman. Go through the Todo API documentation to better understand the API.

The idea for this activity is to interact with the Todo API to perform CRUD operations on the data. You will fetch Todos from their server, add a Todo entry, and update, fetch, and delete that Todo using its ID. Note that this API has a rate limit of five requests per minute for POST, PUT, and DELETE requests and a rate of two requests per second for GET requests.

Sending a POST Request to the Todo API

To add a Todo item to the API, you must send a POST request with the Todo data, such as Todo text, its status, and whether it is complete. The request is sent as JSON data where the todoName key will hold the Todo text that describes the Todo, and the isComplete key tells you whether the Todo is completed or not.

To send a POST request using Postman, you must add the Todo as JSON to the request body, as seen in the following screenshot:

If you send a POST request with the Todo item inside your JSON code, you will get the following response:

{
    "code": 200,
    "data": {
        "todoName": "DC Films Marathon",
        "isComplete": false,
        "createdAt": "2024-03-19T11:24:11.237Z",
        "updatedAt": "2024-03-19T11:24:11.238Z",
        "_id": "65f97aab62da1655623599a4",
        "__v": 0
    }
}
